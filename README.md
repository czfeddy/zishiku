# 知识库项目

这是一个独立维护的知识库功能仓库，用于承载 H5 内容展示、后台内容管理、微信分享能力，以及与现有微信小程序的跳转衔接。

## 项目定位

本仓库是一个单独项目，不属于 `jinxiang` 整体程序仓库，也不是其镜像仓库。

- 当前仓库聚焦“知识库 / 内容分发 / 小程序导流”这项功能
- 可以独立进行代码管理、版本发布、部署和问题追踪
- 允许复用其他项目中的实现思路，但以本仓库为准持续维护

## 目录结构

- `public/`：H5 前台页面、后台管理页面及静态资源
- `server.js`：Node 服务端入口
- `app.py`：Python 服务端实现
- `data/`：站点内容数据与基础存储目录
- `wechat-miniprogram/`：用于承接分享链路的小程序壳
- `docs/`：部署、备案和业务说明文档
- `Dockerfile`、`docker-compose.yml`、`Caddyfile`：部署相关文件

## 功能概览

- H5 首页、详情页、分区页、充值页、成长页
- 后台内容管理
- 微信 JS-SDK 分享
- 微信支付接入预留
- 小程序分享壳与 H5 落地页联动

## 本地运行

### Node 版本

```bash
npm install
npm start
```

默认启动入口为 `server.js`。

### Python 版本

```bash
python app.py
```

## 环境变量

复制 [`.env.example`](./.env.example) 为 `.env` 后再运行项目，不要将 `.env` 提交到 GitHub。

主要配置包括：

- 微信公众号 JS-SDK 配置
- 微信支付配置
- 域名与分享域名配置
- 站点名称与默认展示信息

## 与外部小程序的关系

本项目包含一个用于分享链路验证和页面承接的小程序壳，但它可以指向现有小程序配置。也就是说：

- 本仓库本身是独立项目
- 小程序跳转参数可以复用现有线上配置
- 仓库独立，不影响它所对接的其他业务项目边界
